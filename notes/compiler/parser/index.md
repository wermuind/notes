# Синтаксический анализ

Нисходящий парсер:
+ метод рекурсивного спуска
+ LL-анализаторы

Восходящий парсер:
+ LR-анализаторы (shift-reduce)

### Shift-reduce Parser

<dfn>Shift-reduce parser</dfn> (анализатор сдвига-свёртки) — класс методов
восходящего анализа. На каждом этапе этого прохода синтаксический анализатор
накапливает список поддеревьев или фраз входного текста, которые уже были
проанализированы.

#### Действия:

+ **reduce** (свёртка): если существует правило, такое, что верхние элементы
  стека совпадают с его правой частью, тогда эти элементы стека заменяются на
  левую часть правила; для LR(1) парсера решение принимается не только исходя
  из стека, но и по следующему токену со входа (lookahead)
+ **shift** (сдвиг): взятие очередного токена из необработанного ввода
  (если есть), которое происходит если выполнить сокращение невозможно

Когда токены закончились и более преобразовать текущий стек невозможно,
возникает одна из двух возможных ситуаций:

1. **accept**: последовательность распознана и соответствует языке
2. **error** последовательность нераспознана и не соответсвует языку
